### 🔧 Services Created

#### 1. ActivityLogger (`app/Services/Logging/ActivityLogger.php`)

**Purpose:** Enterprise-grade activity logging

**Methods:**
- `log()` - Generic activity logging
- `logCustomer()` - Customer-specific events
- `logLicense()` - License-specific events
- `logPayment()` - Payment events
- `logSecurity()` - Security events
- `logAuth()` - Authentication events
- `getRecent()` - Fetch recent activity
- `getForSubject()` - Get activity for specific model

**Auto-captures:**
- IP address
- User agent
- Timestamp
- Authenticated user (causer)

**Example Usage:**
```php
$logger = app(ActivityLogger::class);

// Log customer creation
$logger->logCustomer('created', $customer, [
    'plan' => 'single',
    'source' => 'stripe'
]);

// Log license activation
$logger->logLicense('activated', $license, [
    'domain' => 'example.com',
    'ip' => request()->ip()
]);
```

#### 2. AlertService (`app/Services/Monitoring/AlertService.php`)

**Purpose:** System monitoring and alerting

**Methods:**
- `alert()` - Generic alert creation
- `critical()` - Critical alerts (auto-email)
- `error()` - Error alerts (auto-email)
- `warning()` - Warning alerts
- `info()` - Info alerts

**License-Specific Alerts:**
- `licenseExpiring()` - 30-day expiry warning
- `licenseExpired()` - License expired
- `licenseActivationLimitReached()` - Max activations reached

**Payment-Specific Alerts:**
- `paymentFailed()` - Payment gateway failure
- `subscriptionCancelled()` - Subscription cancelled

**Security-Specific Alerts:**
- `suspiciousActivity()` - Security threat detected
- `multipleFailedLogins()` - Brute force attempt

**Features:**
- Auto-logs to Laravel log system
- Sends email to admins for critical/error alerts
- Tracks email delivery
- Resolution workflow

**Example Usage:**
```php
$alertService = app(AlertService::class);

// Critical alert
$alertService->critical(
    'system',
    'Database Connection Lost',
    'Unable to connect to database server',
    ['error' => 'Connection timeout']
);

// License expiring soon
$alertService->licenseExpiring($license);
```

### 🤖 Automated Monitoring

**Command:** `licenses:monitor`

**Signature:**
```bash
php artisan licenses:monitor --send-alerts
```

**Monitors:**
1. **Expiring Licenses** - 30-day window
2. **Expired Licenses** - Past expiry date
3. **Activation Limits** - Max activations reached
4. **Stale Activations** - No heartbeat in 7+ days

**Scheduled:**
- Runs daily at 9:00 AM
- Sends email alerts automatically
- Logs all findings

**Activity Log Cleanup:**
- Runs daily via `model:prune`
- Deletes logs older than 90 days
- Maintains database performance

**Schedule Configuration:** `routes/console.php`
```php
Schedule::command('licenses:monitor --send-alerts')->dailyAt('09:00');
Schedule::command('model:prune', ['--model' => [ActivityLog::class]])->daily();
```

### 🔄 Routes Updated

**Before:**
```php
Route::prefix('admin')->group(function () { ... });
```

**After:**
```php
Route::prefix('admin')
    ->middleware(['auth', 'role:admin'])
    ->name('admin.')
    ->group(function () { ... });
```

**Root Route:**
```php
Route::get('/', function () {
    if (auth()->check()) {
        return auth()->user()->hasRole('admin') 
            ? redirect()->route('admin.dashboard')
            : redirect()->route('customer.dashboard');
    }
    return redirect()->route('login');
});
```

**Auth Routes:** Included via `require __DIR__.'/auth.php'` (Breeze)
